**核心差异点对比：**

1. **append vs extend：**
   - list.append([1, 2])：会将 [1, 2] 当作**一个整体**插入，列表长度只增加 1。
   - list.extend([1, 2])：会将 1 和 2 **拆开**分别插入，列表长度增加 2。
2. **sort vs sorted ：**
   - list.sort()：是列表的方法，**直接修改**原列表。
   - sorted(list)：是内置函数，**返回一个排序后的新列表**，原列表不变。
3. **del vs pop vs remove：**
   - 如果你需要**拿到**被删掉的值：用 pop()。
   - 如果你知道**下标**但不需要返回值：用 del。
   - 如果你不知道下标，只知道**内容**：用 remove()。

#### **元组 (Tuple)**

**元组 (Tuple) 常用操作汇总表**

| 分类             | 功能               | 语法/示例               | 说明与备注                                   |
| ---------------- | ------------------ | ----------------------- | -------------------------------------------- |
| **定义与初始化** | 定义元组变量       | t = (e1, e2, ...)       | **不可修改**，其余特性与列表一致             |
|                  | 定义空元组         | t = tuple() 或 ()       | 初始化一个空元组                             |
|                  | **定义单元素元组** | t = (1 , )              | ==**必加逗号**==，否则会被识别为普通括号运算 |
| **核心特性**     | 不可变性           | -                       | 一旦定义，元素**指向**不可更改               |
|                  | **作字典键 (Key)** | dict = {(1,2): "value"} | 因其不可变且可哈希，常用于复合键             |

---

**核心要点补充：**

1. **“不可变”的本质：**
   - 元组的不可变是指元组所保存的**每个元素的内存地址（引用）不可变**。
   - 如果元组里嵌套了一个列表 t = (1, 2, [3, 4])，虽然你不能把列表换成别的东西，但你可以修改列表内部的值：t[2][0] = 99 是合法的。
2. **元组 vs 列表：**
   - **相同点**：都是有序序列，都支持下标索引、切片、嵌套、包含不同类型。
   - **不同点**：列表是动态的（可变），元组是静态的（不可变）。元组由于结构简单，在处理大量数据时比列表**更省内存**，且查找速度略快。
3. **什么时候用元组？**
   - 当你希望数据**不被意外修改**时（如配置信息、坐标点）。
   - 当需要把一组数据作为**字典的键**时。
   - 函数返回多个值时，Python 默认会将其封装成一个元组。

- **元组解包 (Unpacking) (极其常用！)**：

  ```python
  # 将元组中的元素一次性赋值给多个变量，常用于函数返回多个值时的接收
  a, b, c = (1, 2, 3)  # a=1, b=2, c=3
  ```

- 元组的遍历同列表一致

#### **字符串 (String)**

- 不可修改，支持重复元素且有序



**字符串 (String) 常用操作汇总表**

| 分类           | 功能           | 语法/示例                 | 说明与备注                                   |
| -------------- | -------------- | ------------------------- | -------------------------------------------- |

|                | **修改机制**   | -                         | ==所有修改操作均返回**新字符串**，原串不变== |
| **查询与统计** | 查找下标       | s.find() / s.index()      | find 找不到返回 -1；index 找不到**报报错**   |

| **转换与处理** | 字符串替换     | s.replace(old, new, n)    | 默认替换全部，若指定 n 则只替换前 n 次       |
|                | **字符串分割** | s.split("分隔符")         | 根据分隔符切割，==**返回结果是一个列表**==   |

---

**核心要点补充：**

1. **不可变性 (Immutability)：**
   - 你不能像列表那样通过 s[0] = 'A' 来修改字符串的某个字符，这会直接报错。
   - 任何“看起来”在修改字符串的操作（如 replace, strip），本质上都是在内存中创建了一个**全新的字符串对象**并返回。
2. **find vs index 的选择：**
   - 建议优先使用 find()。因为在实际开发中，如果找不到子串，返回 -1 比让整个程序崩溃报错（index 的行为）要更安全、更容易处理。
3. **split 与 join 是“老搭档”：**
   - split 把字符串拆成列表，join 把列表合回字符串。
   - 示例： **"A,B,C".split(",")  →→ ['A', 'B', 'C'] →→  "-".join(['A', 'B', 'C'])  →→  "A-B-C"。**
4. **规整操作 strip：**
   - strip 不仅可以去空格，还可以去特定字符。例如 "...hello...".strip(".") 结果为 "hello"。它只管两头，不管中间

#### **集合 (Set)**

- 允许修改，不支持重复元素且无序（即不支持下标索引）
- ==集合最常用的功能是：去重 和 逻辑交并差运算==

---

**集合 (Set) 常用操作汇总表**

| 分类             | 功能           | 语法/示例                | 说明与备注                                        |
| ---------------- | -------------- | ------------------------ | ------------------------------------------------- |
| **定义与初始化** | 定义集合变量   | s = {e1, e2, ...}        | **自动去重**，且内部元素**无序**                  |
|                  | **定义空集合** | s = set()                | ==**注意：** 不能用 {}，那是在定义空字典==        |
| **基础增删操作** | 添加元素       | s.add(元素)              | 若元素已存在，则不进行任何操作                    |
|                  | **安全移除**   | s.discard(元素)          | ==**推荐：** 若元素不存在，不会报错==             |
|                  | 强制移除       | s.remove(元素)           | 若元素不存在，会抛出 KeyError 错误                |
|                  | 随机弹出       | s.pop()                  | 随机删除并返回一个元素（因其无序）                |
|                  | 清空集合       | s.clear()                | 移除集合内所有元素                                |
| **数学逻辑运算** | **取差集**     | s1.difference(s2)        | 简写 s1 - s2。返回 s1 有但 s2 没有的元素          |
|                  | **更新差集**   | s1.difference_update(s2) | ==**原地修改：** 直接从 s1 中删掉 s2 也有的元素== |
|                  | **取交集**     | s1.intersection(s2)      | 简写 s1 & s2。返回两个集合共有的元素              |
|                  | **取并集**     | s1.union(set2)           | 简写 s1 \| s2。合并去重，返回新集合               |
| **关系判断**     | 判断子集       | s1.issubset(s2)          | 判断 s1 的元素是否全部都在 s2 中                  |
|                  | 判断超集       | s1.issuperset(s2)        | 判断 s2 的元素是否全部都在 s1 中                  |
|                  | 统计数量       | len(s)                   | 返回集合中不重复元素的总个数                      |



---

**核心要点补充：**

1. **无序性 (Unordered)：**
   - 集合不像列表或字符串，它没有“下标”概念。你不能通过 s[0] 来取值。
   - 因此，集合**不支持 while 循环遍历**（因为没法用索引控制），只能用 **for 循环** 挨个迭代。
2. **唯一性 (Unique)：**
   - 集合最常见的用途就是**给列表去重**。
   - *示例：* list(set([1, 1, 2, 2, 3])) →→ [1, 2, 3]。
3. **元素必须“可哈希”：**
   - 集合的元素必须是不可变的（如数字、字符串、元组）。
   - 你不能在一个集合里放一个列表：{1, [2, 3]} 会报错。
4. **discard vs remove：**
   - 在不确定元素是否一定存在时，永远优先使用 discard()，它可以避免程序因为一个小小的删除动作而崩溃。

#### 字典 (Dictionary / Hash Table)

- 类似哈希表，存储**键值对 (Key-Value Pair)**，通过键极速寻址对应的值。
- 允许修改，不支持重复key且无序（Python 3.7+内部保持插入顺序）。
- ==字典的 Key 必须是不可变类型（如数字、字符串、元组），不能是列表或字典==。



**字典 (Dict) 常用操作汇总表**

| 分类             | 功能              | 语法/示例             | 说明与备注                                                   |
| ---------------- | ----------------- | --------------------- | ------------------------------------------------------------ |
| **定义与初始化** | 定义字典变量      | d = {k1: v1, k2: v2}  | 键(Key)必须**不可变且唯一**，值(Value)可重复                 |
|                  | 定义空字典        | d = {} 或 dict()      | 初始化一个没有任何键值对的字典                               |
| **新增与修改**   | **动态更新/新增** | dict[key] = value     | ==**核心逻辑：** 键不存在则新增，键存在则覆盖旧值==          |
|                  | 批量更新          | d1.update(d2)         | 将字典 d2 的键值对合并到 d1 中                               |
| **安全获取**     | **安全取值**      | dict.get(key, 默认值) | ==**防崩溃：** 键不存在时返回默认值（None），不报 KeyError== |
|                  | **设置默认值**    | dict.setdefault(k, v) | ==若键不存在则存入该键值并返回 v；若存在则返回原值==         |
| **删除操作**     | 弹出键值          | dict.pop(key)         | 删除指定键并**返回其对应的值**                               |
|                  | 内存删除          | del dict[key]         | 直接从字典中移除指定键值对                                   |
|                  | 清空字典          | dict.clear()          | 移除所有键值对，得到 {}                                      |
| **视图与遍历**   | 获取所有键        | dict.keys()           | 返回所有 Key 的集合视图                                      |
|                  | 获取所有值        | dict.values()         | 返回所有 Value 的集合视图                                    |
|                  | **获取键值对**    | dict.items()          | ==**最常用：** 返回 (Key, Value) 元组构成的列表视图==        |
|                  | 统计数量          | len(d)                | 返回字典中键值对的总数                                       |



---

**核心要点补充：**

1. **“键 (Key)” 的硬性要求：**
   - **必须唯一**：如果出现重复的键，最后存入的那个会覆盖前面的。
   - **必须不可变 (Hashable)**：字符串、数字、元组可以做键；**列表、字典、集合不能做键**（因为它们是可变的，哈希值会变）。
2. **dict[key] vs dict.get(key)：**
   - 在算法题中，如果你**确定**键一定存在，用 dict[key]。
   - 如果你**不确定**（如统计单词次数），永远用 dict.get(key, 0)，这样可以优雅地处理第一次出现的情况。



- **字典的遍历操作：**

```python
# 1. 只遍历 Key (默认就是遍历key)
for key in my_dict:
    print(key, my_dict[key])

# 2. 同时遍历 Key 和 Value (最常用)
for key, value in my_dict.items():
    print(f"键是 {key}, 值是 {value}")
```

## **文件读取**

**什么是文件呢？一篇文章、一段视频、一段音频、一个可执行程序都可以被保存为文件。操作系统以文件为单位管理磁盘中的数据，一般来说文件可分为：==文本文件、视频文件、音频文件、图像文件、可执行文件等等，并且以不同后缀来区分==**

---

### **Open**

- 使用 open() 可以打开或创建一个文件

> [!CAUTION]
>
> 语法： open(name, mode, encoding)
>
> name：是文件名，可以包含路径
>
> mode：设置打开文件的模式：r (读) ，w (写，原有内容会被覆盖) ，a (追加)
>
> r+  w+  a+ 为读写模式
>
> encoding：文件编码格式（常用 "UTF-8"）
>
> eg: f = open('Python.txt', 'r', encoding="UTF-8")



### **读取操作**

- file.read(num)：num表示读取的字节数，无参读取所有数据，**返回一个长字符串**
- file.readlines()：按行的方式一次性读取文件内容，**返回一个列表**，每行为一个元素
  ==连续 read/readlines 同一文件时，下一次读取会延续上一次读取的指针结尾来读==
- for循环读取文件行 (推荐，省内存)：

```python
for line in open('file.txt', 'r'):
    print(line)
```

- file.close(): 关闭程序 ，否则会持续占用文件  ==内置flush函数刷新功能==
- **with open 语法 (最佳实践)**： 可以在操作完成后自动 close 文件，防止忘记关闭。

```python
with open('file.txt', 'r') as f:
    for line in f:
        print(line)
```

- file.write("内容") ，直接调用时，内容不会立刻写入硬盘，而是先放在内存缓冲区，调用 flush() 或 close() 时才会真正写入硬盘。

## **异常**

- 异常具有传递性（底层的报错会一层层往上抛）

### **捕获异常**

#### **捕获常规/所有异常**

```python
try:
    可能出错的代码 
except Exception as e:     # 捕获所有类型的异常，e 记录异常的具体信息
    出现异常执行的代码
    print(f"报错原因：{e}")
```

#### **捕获指定异常或多个异常**

```Python
try:
    可能出错的代码 
except NameError as e:     # 只捕获特定异常
    出现NameError执行的代码
except (TypeError, ValueError) as e:  # 用元组捕获多个指定异常
    出现这两种异常执行的代码
```

------

### **异常 Else 与 Finally**

- **else**： 可以选择写或不写，逻辑是在 try 中**未触发任何异常后**执行（即一切顺利时执行的代码）。
- **finally**： 是**无论异常与否**最后都必定会执行的语句（常用于资源释放，如关闭文件或数据库连接）。

## **模块与包**

- 模块(Module) 是一个**Python文件**，是一个工具包，里面有类、函数等，供导入使用

### **模块的导入**

- 导入语法： [from 模块名] import[模块 | 函数 | 类 | 变量 | * ] [as 别名==(适用于长名)==]
  ==若 import 模块 ，此时使用函数时应：如 time.sleep()==
- 使用 from 模块 import 功能名  表示只使用该模块的该功能
- from 模块 import *  表示导入该模块的全部功能
  ==在 from 情况下，使用函数便可直接使用，如 sleep(5) , 而第一种情况则需要用 模块名.功能名==

### **自定义模块与包**

1. 创建一个python文件（如 tools.py），写好函数后，在另一个文件里即可 import tools 调用。

2. **测试代码隔离**：若自定义模块中有测试代码，别的文件导入时会自动执行测试代码。可通过以下判定来确保只有直接运行该文件时才测试：

   ```Python
   if __name__ == '__main__':
       测试代码
   ```

3. **控制导出范围**：当其他文件使用 from module import * 时，可以在自定义模块中使用变量 __all__ = ['函数1', '函数2'] ，只有在列表中的函数才能被默认导出。

4. **包 (Package)**：就是存放了**一堆模块**和**一个 __init__.py 文件**的文件夹。调用格式： import package.module。

5. **第三方包安装**：使用 pip，建议配置国内源加快速度：
   pip install -i https://pypi.tuna.tsinghua.edu.cn/simple 包名

## **面向对象**

### **类与对象**

- 设计类： class 类名:
- 创建类属性与初始化：
  ==Python中，类内定义函数 __init__(self, element1, ...) (构造函数) 来初始化成员变量==
- 定义类行为（成员方法）：
  ==定义方法时，第一个参数必须是 **self** 。在方法内访问类属性或调用其他方法时，必须使用 self.属性名，否则会被当成局部变量==。

### **魔术方法**

- Python类内置的、具有特殊触发机制的方法（以双下划线包围，如 __init__）被称之为**魔术方法**。

- **__str__ 方法**：可以按想要的形式去修改 print(对象) 时的字符串输出。

  ```Python
  def __str__(self):
      return f"这是一个拥有指定格式的类实例"
  ```

- **比较大小方法**：类似于**运算符重载**。

> [!IMPORTANT]
>
> 常用比较魔术方法
> __lt__：实现小于操作符 <
> __gt__：实现大于操作符 >
> __le__：实现小于等于操作符 <=
> __ge__：实现大于等于操作符 >=
> __eq__：实现等于操作符 ==
> __ne__：实现不等于操作符 !=

### **封装**

- 作用：不对外暴露类的内部数据和细节。实现特性：定义**私有成员和方法**。
- 语法：在成员属性名或方法名前加双下划线 __ (例如 self.__money = 0)。
- ==私有成员和方法只能在**类内部**的方法中被使用，类外部实例化出的对象无法直接调用。==

------



### **继承与复写**

- 作用：在已有的类基础上扩展或修改行为，实现代码复用。

#### **继承父类**

- 语法： class Son(Father1, Father2):  （单继承与多继承）
- **pass 关键字**：在类或函数中，如果暂时没有内容，使用 pass 占位以**符合语法规范**。
- ==若多继承的多个父类中有**同名**成员变量或方法，**写在左边（先继承）**的父类优先级更高==。

#### **复写 (Override) 与调用父类**

- 当对父类内的某些成员**有修改需求**时，直接在子类中定义一个**同名**的方法或属性即可覆盖。
- **调用被复写的父类方法**：如果在子类中还想执行原有父类的逻辑，可以使用 super()。

```Python
class Phone:  
    def print_info(self):  
        print("hello father")  
  
class SunPhone(Phone):  
    def print_info(self):  
        print("hello son")  
        super().print_info()  # 现代写法：自动调用父类的同名方法
        # 或者旧写法: Phone.print_info(self)
```

------



### **多态**

- 概念：同样的行为（方法调用），针对不同类型的子类对象，表现出不同的执行状态。
- **抽象类与抽象方法**：
  - 如果父类的方法体只是空实现（pass）或者直接 raise NotImplementedError，这种方法称为**抽象方法**。
  - 包含抽象方法的类充当**接口**的角色（做顶层规范设计），强制要求所有继承它的**子类必须自己去重写并实现**这个具体的逻辑。

### **类型注解**

- 作用：帮助 IDE (如 PyCharm) 对变量、参数或返回值做类型推断，提供代码补全和语法检查。
- ==注：Python 是动态语言，类型注解仅仅是“提示性”的，即使实际传入类型不符，也不会导致程序运行崩溃==。

------



#### **变量注解**

- 基础语法： 变量: 类型 = 值 (如 value: int = 10)
- 类对象注解： element: ClassName = ClassName()
- **数据容器注解**：
  - 简易注解： my_list: list =[1, 2, 3]
  - 详细注解： my_list: list[int] =[1, 2, 3]
  - ==详细注解时：元组需要按位置标注每一个元素类型（如 tuple[int, str, bool]）、字典需要两个类型分别代表键和值（如 dict[str, int]）==。

------



#### **函数注解**

- 给形参和返回值进行注解，极大提升函数的代码可读性。
- 语法： def function(形参1: 类型, 形参2: 类型) -> 返回值类型:

------



#### **Union 注解**

- 当变量允许是多种数据类型之一，或者容器内包含多种混合类型时，使用 Union。

```Python
from typing import Union

# 表示这个列表里的元素既可以是 str 也可以是 int
my_list: list[Union[str, int]] =[1, 'ch', "hello"]

# 用在函数中：既可以接收整型也能接收浮点型
def compute(num: Union[int, float]) -> Union[int, float]:
    pass
```
