# **存储引擎**

> 存储引擎就是存储数据、建立索引、更新/查询数据等技术的实现方式
> 
> 不同的表可以有不同的存储引擎，因而存储引擎也可以称为表类型

-  建表时指定存储引擎( 未指定时默认是**InnoDB**)

```
 create  table  表名 (
			... 
 ) ENGINE = InnoDB ;

```

<mark style="background: #ABF7F7A6;">InnoDB特点
</mark>

- DML操作遵循ACID模型，支持事务

- 行级锁，提高并发执行性能

- 支持外键约束，提高表数据的正确性和完整性

<mark style="background: #ADCCFFA6;">MylSAM特点</mark>

- 不支持事务、外键

- 支持表锁，不支持行锁

- 访问速度快

<mark style="background: #D2B3FFA6;">Memory特点</mark>

- 数据存储在内存中，受硬件影响，只能将这些表作为==临时表或缓存==使用

- 支持hash索引（默认）

# **索引**

> 一种用来帮助MySQL高效获取数据的数据结构（有序）

| 索引结构             |     | 描述                              |
| ---------------- | --- | ------------------------------- |
| B+Tree 索引        |     | 最常见的索引，大部分引擎都支持该索引              |
| Hash 索引          |     | 对精确匹配索引列的查询才有效，不支持范围查询          |
| R-Tree (空间索引)    |     | 空间索引是MylSAM的一种特殊索引，主要用于地理空间数据类型 |
| Full-text (全文索引) |     | 是一种通过倒排索引，快速匹配文档的方式             |
![[索引支持.png]]

---聚集索引下存储的是该行数据，二级索引下一般存储的是主键

---回表查询：例如select \* from table where name = 'Arm' 首先查二级索引，再查聚集索引


![[索引分类.png]]

## **索引语法**

-  创建索引： create \[ unique | fulltext ] index 索引名 on 表名 ( 字段名 )

-  查看索引：show index from 表名

-  删除索引：drop index 索引名 on 表名
